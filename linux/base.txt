中继器：信号的复制、调整和放大功能,以此来延长网络的长度
集线器(Hub)：多个接口，连接主机，通过广播方式转发数据帧
网桥(Bridge)：两个接口，连接集线器，缓存两边接口的mac地址，分隔集线器之间的广播域
交换机(Swich)：多个接口，连接主机，缓存每个接口的mac地址，通过单播转发数据帧

连接网线后确认网线指示灯有在正常闪烁，表示网络硬件连接正常

数据链路层：即插即用,可以看作是有特殊功能的网线
	mac地址：物理地址，网线两端传递，多段传输mac地址会一直变
	mac地址表：mac地址和接口的映射
	交换机：
		微型电脑，有cpu(处理)和内存(存储)  
		有进程
		交换机中可能有多种内存（接口--mac地址）
	背板带宽，是交换机接口处理器或接口卡和数据总线间所能吞吐的最大数据量
	vlan: 同一台分隔广播域，access访问模式
	trunk: 不同交换机之间的相同vlan(每台都需要创建)通讯，原理是两交换机trunk接口之间在帧上添加vlan头，trunk模式（接口和mac地址的映射是一对多）

# https://www.cnblogs.com/onepixel/articles/10256314.html
# 在每台主机中都有一张ARP表，ARP协议是工作在网络层的协议，通过广播帧方式，将IP地址解析为MAC地址。

网络层：分网段转发--方便寻址
	ip地址：逻辑地址
	ip地址和子网掩码进行逻辑运算得出网段（二进制：子网掩码为1的对应ip部分照抄 + 子网掩码为1的对应ip部分为0）
	ping 127.0.0.1 或 ping 本机网卡ip : 为了保证本机ip协议簇没问题
	ping 非本机段ip ：查看路由是否有问题
	# 经验：192.168.1.0网段(首) + 192.168.1.255广播地址(尾) + 可用地址1-254(一般用2-253，首尾地址之一作为网关)
	路由器是根据路由表转发的，路由表上记录着路由条目
	路由条目：数据包出去怎么走，无法控制进来的数据包(目的地址可以是一个ip，但不会这样做)
		default via 192.168.50.2 dev ens33 	#默认路由
		192.168.50.0/24 dev ens33           #直连路由，同一网段，插上网线就会自动生成,不需要网关  
		192.168.60.0/24 via 192.168.60.2    #非直连路由，通过网关出去

排错思路：
	查看硬件指示灯或更换硬件，确定本机硬件没问题
	ping 127.0.0.1 或 ping 本机网卡ip : 为了保证本机ip协议簇没问题
	每段网线直连设备互ping(本机段其它ip) ：查看设备之间是否有问题
	ping 非本机段ip ：查看路由是否有问题
